Use esta caracteristica solo si sabe lo que esta haciendo. Cualquier error corrompera el archivo de script haciendolo ilegible por el juego o un editor de scripts.

Sanny Builder admite escribir un contenido sin formato en un archivo de salida. Todos los valores dentro de este constructo estan escritos en un archivo de salida sin ningun control.

Sintaxis:
hex
<cualquier numero hexadecimal>
end

Ejemplo:

hex
04 00 02 08 00 04 01
end

Esta secuencia de bytes es un codigo de operacion compilado 0004: $2 = 1, por lo que el juego lo leera correctamente y establecera el valor de la variable $2 a 1.

Debe especificar cada byte que va a escribir con dos digitos. Todos los espacios son ignorados. El compilador procesa dos digitos con un espacio entre ellos como valor de un solo byte. Un unico digito (0...F) se pega con 0. Entonces, por ejemplo, A B C se escribe como un numero 0xAB0C.

El constructo HEX..END tambien acepta literales de cadena, etiquetas, identificadores de variables globales, nombres de modelos. Se compilan sin un tipo de datos precedente.

:get_offset
hex
04 00 02 $PLAYER_CHAR 01 @get_offset
end

Esta es una copia exacta del opcode 0004: $PLAYER_CHAR = @get_offset

Los literales de cadena se compilan como una secuencia de caracteres entre comillas dobles.

hex
"Esta es una cadena"
end

Las siguientes secuencias de escape son compatibles con una cadena:

  • \0 - nulo - 0
  • \b - retroceso - 8
  • \t - tabulacion - 9
  • \n - avance de linea - 10
  • \r - retorno de carro - 13
  • \x<nn> - secuencia de escape numerica- nn
  • \<char> - Char de escape- <char>

Ejemplo:

hex
"\0\b\t\n\r\xDD"
end

Produce la siguiente secuencia de bytes: 00 08 09 0A 0D DD.
Actualmente, multiples espacios en un literal de cadena se convierten en uno solo. Por lo tanto, una linea "Esto    es  una cadena" se convierte en "Esto es una cadena". Use un caracter de escape \ para crear multiples espacios en una cadena literal: "Esto\ \ \ es \ una \ \ \ \ cadena".

Tambien puede usar el tipo de datos aDMA para escribir una constante numerica en un archivo de salida. El numero despues del signo & puede ser positivo o negativo, decimal o hexadimal.

hex
&1000 &-0xA33500
end
Este ejemplo produce la siguiente secuencia de bytes: E8 03 00 CB 5C FF.
