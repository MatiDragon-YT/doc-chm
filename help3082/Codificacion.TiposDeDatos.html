Ciertos elementos de código deben comenzar o terminar con caracteres específicos. Ayudan al compilador a reconocer su significado. La siguiente lista contiene todos los tipos de datos conocidos por el compilador.

Variables

Una variable global comienza con $ seguido de un identificador válido (cualquier combinación de letras, dígitos y _).

0004: $MiVariable = 100

Una variable local como nombre solo puede tener un número seguido de @. El número es el índice de variable local en la lista de variables locales asignadas a este script o una misión. 

0006: 100@ = 10

Una variable ADMA (Advanced Direct Memory Access) es una referencia al desplazamiento en el archivo main.scm. Sirven como variables globales, es decir, se puede leer y escribir en una dirección en main.scm.

$MiVariable = &0 // leer los primeros cuatro bytes del main.scm y escribirlos en $ MiVariable
&57 += &120(&231, 4i) // se puede utilizar como elemento de un arreglo

Las variables ADMA no afectan al tamaño del espacio de variables globales en el encabezado main.scm.

Etiquetas

Las etiquetas se utilizan para hacer referencia a la ubicación del código desde saltos incondicionales (opcodes 0002, 0050)o condicionales (opcode 004D). Comienzan con : seguido de un identificador válido (un nombre de etiqueta).

:MiEtiqueta

Para hacer referencia a la etiqueta desde un opcode, utilice el @ y, a continuación, escriba el nombre de la etiqueta.

0002: jump @MiEtiqueta

Si el nombre de una etiqueta se escribe como una declaración independiente seguida de () representa una llamada a la subrutina con el comando gosub:

// Las siguientes declaraciones son equivalentes
MiEtiqueta()
gosub @MiEtiqueta
0050: gosub @MiEtiqueta
Los anidamientos estan limitados a 8 y cada uno debe contener un return obligatorio.

Cadenas Literarias

Un texto entre comillas simples ' ' es un literal de cadena corta (15 caracteres máx.)

03A4: name_thread 'MAIN'

También se permiten cadenas en blanco: ' '.
Un texto entre " " es un literal de cadena larga (la longitud máxima viene determinada por el código de operación que utiliza)
Si el literal contiene " debe escribir \ antes de él.

0662: write_debug_message "Hello, \"world\"! \n 'Here we go!'"

También se permiten cadenas en blanco: " ".

Variables de Cadenas

Cadena Local Global
Corta

Una variable local que contiene una cadena corta se inicia con @s.

05AA: 1@s = 'LOCAL'

Una variable global que contiene una cadena corta se inicia con s$.

05A9: s$MyString = 'GLOBAL'
Larga

Una variable local que contiene una cadena larga se inicia con @v.

06D2: 1@v = "LONG_LOCAL"

Una variable global que contiene una cadena larga se inicia con v$.

06D1: v$MyString = "LONG_GLOBAL"

Nombre de Modelo

Los modelos definidos en los archivos .ide, pueden ser referenciados con # seguido de su nombre.

0247: request_model #CELLPHONE

Números Hexadecimales

0x - un número hexadecimal
-0x - un número hexadecimal negativo

0004: $variable = -0xBB08

Un número hexadecimal debe estar dentro del rango -80000000..7FFFFFFF.
