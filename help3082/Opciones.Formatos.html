Aqui puede configurar los nombres de los archivos de origen, el formato de las etiquetas y los identificadores..

Alli estan las mascaras de nombre de archivo. La linea superior contiene la ruta y el nombre del archivo de texto que se crea al descompilar. Las lineas inferiores contienen rutas y nombres de dos archivos binarios SCM e IMG que se crean durante la compilacion. Todas las lineas tienen la misma sintaxis Utiliza palabras especiales que significan parte del nombre completo del archivo:

$dir – ruta de archivo
$name – nombre del archivo
$ext – extension de archivo

Example, if you decompile the file C:\MyDir\main.scm, entonces

$dir = C:\MyDir;
$name = principal
$ext - .scm
Entonces, si la mascara es $dir\$name.txt, el archivo de salida sera nombrado C:\MyDir\main.txt

Aqui puede elegir uno de los diversos formatos de nombre para las etiquetas utilizadas en la descompilacion.

Global Offset: all labels tendra un nombre numerico, por ejemplo @12345. El numero significa la compensacion de esta etiqueta en el archivo que se descompila.

Thread+Local Offset: El nombre de la etiqueta constara de dos partes: un nombre del hilo donde esta la etiqueta (se usa el codigo de operacion name_thread para esto) y adicionalmente se compensa con el comienzo de este hilo, por ejemplo @MAIN_12.

Thread + Global Offset: esta opcion es similar a la anterior, excepto que despues del nombre del hilo no habra un desplazamiento local sino el global (como el primer caso), por ejemplo @HELP_34567.

Si desea que los desplazamientos sean hexadicos, seleccione la opcion adecuada. Por defecto, todas las compensaciones son decimales.

Estas opciones hacen que el decompilador use o no los nombres personalizados para varios literales de scripts.

Los nombres de las etiquetas se encuentran en el archivo CustomLabels.ini en la carpeta SB \ data \ <juego>. Este archivo contiene la lista de nombres y compensaciones de etiquetas (para main.scm version 1.0). Si se marca la primera casilla, el decompilador la etiqueta como se define.

Los nombres de variables estan presentes en el archivo CustomVariables.ini para cada juego. Parece la lista de direcciones de variables globales (DMA) y su representacion textual. Si esta casilla esta marcada, el descompilador le da a la variable global su nombre personalizado.

La lista de nombres de matrices esta presente en el archivo CustomArrays.ini. Su sintaxis es la siguiente: el primer numero es el nombre de la variable global (DMA) que es el primer elemento de la matriz (nombre de la matriz), el segundo - el tamanio de la matriz, el tercero - el nombre personalizado. Esta opcion se usa para que el descompilador agregue un indice a algunas variables globales que son una matriz. Entonces, estas variables se vuelven indexadas.

Esta opcion tiene dos significados. El decompilador nombra variables globales, etiquetas, matrices utilizando el caso de letras seleccionado. El compilador escribe las cadenas en el SCM usando el caso de letra seleccionado tambien.
