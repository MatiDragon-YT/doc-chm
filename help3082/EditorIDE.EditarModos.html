Sanny Builder es compatible con muchos juegos y plataformas, por lo que hay una configuración predefinida para cada juego compatible llamado modo de edición.

Los modos de edición proporcionan a Sanny Builder la siguiente información:

  • un juego objetivo
  • rutas a los archivos con los datos del juego
  • rutas a archivos con información de soporte(etiquetas, variables, constantes, etc.)

    • De forma predeterminada, Sanny Builder lee la configuración de los modos desde el archivo SB/data/modes.xml. Esta trayectoria se puede personalizar con la opción de la consola, con -x .

    El archivo modes.xml está abierto para su modificación y extensión, y los usuarios pueden crear sus propios modos cambiándolo.

    Formato del archivo

    El modes.xml es un archivo en formato XML y se puede manipular en cualquier editor de texto. Define los modos disponibles.

    El nodo raíz es modes y no tiene atributos. El archivo solo puede tener un nodo raíz.

    Cada modo de edición es un nodo secundario de modes que comienza con la etiqueta de apertura mode y termina con la etiqueta de cierre /mode. El elemento mode tiene atributos obligatorios y opcionales como se describe a continuación. El contenido de mode es un conjunto de etiquetas específicas (propiedades) que definen rutas de acceso a directorios o archivos.

    Atributos de modo

    ID

    id es un identificador obligatorio y único del modo. Sanny Builder utiliza el id para guardar algunos ajustes de usuario para este modo, por ejemplo, un directorio de juego.

    Un valor válido para este atributo es una serie única de caracteres que no se utilizan para el identificador de ningún otro modo.

    EXTENDS

    Un modo puede extender otro modo (el principal) para reducir el número de propiedades duplicadas. Es útil para diferentes versiones de un juego donde la mayor parte de la configuración es la misma, excepto por algunas propiedades. El padre también puede extender otro modo.

    Cuando falta una propiedad, Sanny Builder recorre recursivamente todos los modos primarios que intentan encontrar la propiedad.

    Un valor válido para este atributo es el identificador de otro modo definido en el mismo archivo.

    TITLE

    title define el nombre mostrado del modo. Debido a las restricciones de interfaz evitar nombres largos y mantenerlo dentro del límite de 24 caracteres.

    GAME

    El atributo game define un juego de destino para el modo. Hay 6 valores válidos:

  • gta3
  • vc
  • sa
  • lcs
  • vcs
  • sa_mobile

    • Antes de desmontar o compilar un script, asegúrese de que el modo de edición correcto esté activo. Cada juego tiene un formato de script único y los scripts compilados para un juego no son compatibles con scripts para otro juego. Incluso si el script se compila sin errores, el juego se bloquearía al intentar leer un script en un formato diferente.

    Un archivo de script compilado puede almacenar información sobre qué juego está hecho para. Al abrir un script de este tipo, Sanny Builder le pide que cambie el modo por el correcto. Ignorar este mensaje puede provocar un bloqueo del desensamblador, porque el formato de script es inesperado para él.

    Sanny Builder muestra un icono de juego delante del nombre del modo de edición para que conozcas el juego objetivo.

    TYPE

    Un modo para cada juego de destino debe ser predeterminado. Esto significa que Sanny Builder utiliza este modo cuando se ejecuta con la opción de CLI --game

    El valor válido para este atributo es default. Omita este atributo para los modos no predeterminados.

    Puede agregar un nuevo modo editando manualmente el archivo games.xml ubicado en el directorio <SB>\data\.

    Para cambiar el modo de edicion, haga clic en en la esquina inferior derecha de la ventana principal de Sanny Builder. Aparecera una lista de los modos disponibles. Elija uno y haga clic en el. Sanny Builder volvera a cargar todos los archivos necesarios y puede comenzar a trabajar.

    Antes de desmontar o compilar un script, asegurese de que el modo de edicion correcto este activo. Incluso si el script se compila sin errores, el juego se bloqueara cuando intente leer un script en un formato diferente.

    Un archivo de script compilado puede tener una informacion para que juego esta hecho. Por lo tanto, si descompila una secuencia de comandos de ese tipo en otro modo de edicion que este activo, Sanny Builder le propondra que cambie la modalidad a la correcta. Si se rechaza este aviso, puede producirse un bloqueo del desensamblador, ya que el formato del script es inesperado.
