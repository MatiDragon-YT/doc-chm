Funciones

El modulo base CLEO.asi version 4.1 tiene la capacidad de agregar dinamicamente controladores opcode gratuitos (0AF0...7FFF). El registro de un nuevo controlador se puede hacer desde el complemento CLEO, que permite a los desarrolladores externos escribir nuevos codigos de operacion, ampliando asi la funcionalidad de las secuencias de comandos CLEO y GTA en general.

El SDK de CLEO incluye

  • Archivo de ayuda (en ruso)
  • Archivos CLEO.h, CLEO.lib utilizados para vincular con el modulo base CLEO.asi
  • El codigo fuente de los complementos de demostracion IniFiles.cleo, FileSystemOperations.cleo e IntOperations.cleo

Aplicacion

Formato de controlador de codigo de operacion

El manejador de codigo de operacion tiene el siguiente formato:

OpcodeResult WINAPI opcodeHandler(CScriptThread* thread);

Descripcion:

thread puntero a la estructura de flujo.

Valor de retorno:
La funcion debe devolver uno de los valores:

OR_CONTINUE indica que continue la ejecucion del script actual.
OR_INTERRUPT pasa la ejecución al siguiente script de la cola.
