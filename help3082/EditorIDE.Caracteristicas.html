Sanny Builder otorga al usuario final las siguientes posibilidades

Resaltado de sintaxis

Sanny Builder resalta un codigo para facilitar la comprension. El esquema de color de resaltado es modificable mediante las opciones.

Marcadores / Salto rapido

Puede marcar la linea presionando Ctrl+Shift+0..9 y luego pasar a este presionando Ctrl+0..9. Para eliminar todos los marcadores, use la opcion Editar -> Borrar todos los marcadores. El segundo metodo es a traves de una opcion 'Ir a linea' invocada presionando Ctrl+G. Ingrese un numero de secuencia de la linea necesaria y el editor movera el cursor hacia el.

Busqueda instantanea

La funcion InstaSearch permite encontrar una palabra o una etiqueta directamente en el editor sin usar el cuadro de dialogo Buscar.

Para encontrar otra aparicion de una palabra debajo del cursor, presione Ctrl + Num6 para comenzar a buscarlo a continuacion en un codigo, o Ctrl + Num4 buscar arriba. El cursor se movera inmediatamente a la linea donde se encuentra dicha palabra. Sigue presionando Ctrl + Num4/Num6 para encontrar otras ocurrencias de esta palabra.

Para saltar a la linea con una etiqueta (por ejemplo: MyLabel), coloque el cursor sobre una referencia de etiqueta (@MyLabel) y presione Ctrl + Num2. El editor movera el cursor a la linea con esta etiqueta.

Si la posicion del cursor sigue siendo la misma, significa que InstaSearch no encontro la palabra necesaria en un codigo.

Antes de cambiar la posicion del cursor, InstaSearch recuerda donde esta ahora, por lo que si desea regresar, presione Ctrl + Num8.

Visualizacion de diversa informacion sobre Opcode

SB muestra una serie de parametros para el codigo de operacion, un valor digital del modelo debajo del cursor en la barra de estado cuando la opcion Mostrar informacion del codigo de operacion esta habilitada.

Conversor de codigo fuente de SA Mission Builder

Sanny admite la sintaxis de SAMB 0.33. Para usar el convertidor, abra el archivo fuente en Sanny Builder y seleccione el elemento del menu: 'Herramientas -> Conversor de codigo -> MB -> SB'. Tenga en cuenta que el convertidor utiliza el archivo 'MB.ini' del directorio SB que contiene las variables utilizadas por SAMB y sus direcciones de memoria. Si agrego sus propias variables en el archivo variables.ini (del SAMB 0.33), copielas en el archivo MB.ini.

Busqueda de codigo de operacion

Use una herramienta integrada de busqueda de codigo para encontrar el codigo de operacion que necesita.

Gestion de Coordinaciones del Jugador

Presione F4 cuando se esta ejecutando SA y aparecera la ventana Administrador de Coords. Aqui puede establecer nuevos coords de jugadores y copiar los actuales. En la linea de edicion superior, puede ingresar las tres coordenadas XYZ, delimitadas por un solo espacio o ",". Ademas, puede insertar coords actuales de CJ en el texto presionando Ctrl+Shift+C. Para insertar el angulo del jugador, presione Ctrl+Shift+E.

Macros Support

Una macro es el grupo de lineas que se insertan en el editor cuando se escribe un nombre de macro (por ejemplo, cargar) y se presiona el F2 boton. El editor agregara la siguiente parte del codigo:

#.Load

while not #.Available

  wait 0

end

Todas las macros estan contenidas en el archivo 'macroes.txt'. La sintaxis de este archivo es la siguiente:
Un nombre de macro se escribe como una sola linea. El signo igual (=) debe seguir el nombre del macro. Despues del signo igual, tambien puede escribir una breve descripcion de la macro. Las lineas de codigo de macro siguen su nombre, cada linea comienza con el signo de igual. La posicion del cursor esta marcada con | personaje

Ademas, es posible agregar una macro en el archivo macroes.txt directamente desde el editor. Para este proposito, seleccione una parte del codigo y haga clic en el menu "Servicio -> Agregar macros". Escriba el nombre de una nueva macro, la descripcion (opcionalmente) en la ventana de dialogo y presione el boton OK. La macro esta lista para ser utilizada.

Para llamar a la lista de macros, presione Ctrl+J.

Tambien puede grabar su secuencia de presionar la tecla (macros) y reproducirla mas tarde. Considera el siguiente codigo:

$Actor = Actor.Create(CivMale, #MALE01, 100.0, 100.0, 10.0)
$ActorWithGun = Actor.Create(CivMale, #MALE01, 110.0, 100.0, 20.0)
$Gang01 = Actor.Create(CivMale, #MALE01, 120.0, 100.0, 30.0)
$Gang02 = Actor.Create(CivMale, #MALE01, 130.0, 100.0, 40.0)
$Killer = Actor.Create(CivMale, #MALE01, 140.0, 100.0, 50.0)
$ActorWithoutGun = Actor.Create(CivMale, #MALE01, 150.0, 100.0, 60.0)

Supongamos que necesita intercambiar las asas del actor en cada par (es decir, $ActorWithGun en lugar de $Actor, y viceversa).

Coloque el cursor en la primera linea y presione Ctrl+M. El editor comienza inmediatamente a registrar todas las pulsaciones de teclas, asi que ten cuidado!

- presione y mantenga Ctrl y presione una vez →
- Ahora presiona Shift+Home y Ctrl+Ins.
- Presione ↓ . El cursor debe estar en la segunda linea con la variable global en el portapapeles
- Presione Ctrl + → y Shift+Ins.
- Presione Ctrl + ←, Shift+Home, Ctrl+Ins y Delete
- Presione ↑
- Presione Ctrl + Shift + → y Shift+Ins
- Presione Home se ira a arriba.

Ahora las primeras dos lineas se ven asi:

$ActorWithGun = Actor.Create(CivMale, 100.0, 100.0, 10.0)
$Actor = Actor.Create(CivMale, 110.0, 100.0, 20.0)

y el cursor esta en el comienzo de la primera linea. Ahora presiona Ctrl+M dejar de grabar.

Ahora puede repetir las pulsaciones de teclas grabadas presionando Ctrl+P. Coloque el cursor en la tercera linea, presione Ctrl+P y los manejadores de actores seran intercambiados.

Mientras procesa, puede pausar / reanudar la grabacion presionando Ctrl+P.

Sanny Builder permite usar un nombre de mision en el codigo de operacion 'start_mission'. El nombre de la mision es la etiqueta definida con un comando DEFINE MISSION. Por ejemplo, tienes:

DEFINE MISSION 10 AT @MYMISSION

En lugar de start_mission 10 puedes usar start_mission MYMISSION.

Tambien esta la opcion Reemplazar numeros de misiones. Cuando esta habilitado, el descompilador reemplaza automaticamente todos los numeros de mision con sus nombres.

Para llamar a la lista de nombres de mision, presione Ctrl+Space. El cursor debe estar directamente despues del comando start_mission.

Ademas, puede usar InstaSearch para saltar a la mision desde su nombre.

Nombres de misiones personalizados

Dado que v3.2.0 Sanny admite nombres de mision personalizados para cada modo de edicion disponible. Los nombres de las misiones se almacenan comunmente en un archivo missions.txt ubicado en una carpeta de data\<juego>. El descompilador usa este archivo para comentar cada entrada de mision con su titulo. Si un archivo main.scm contiene algunas misiones personalizadas, puede editar este archivo para tener los titulos adecuados despues de la descompilacion.

Interfaz multilingüe

Sanny Builder tiene una interfaz totalmente multilingüe traducida a 15 idiomas. Puede cambiar los idiomas en las opciones (F10). El archivo 'lang\Como traducir.txt' contiene mas informacion sobre como crear su propia traduccion.

Herramientas externas/menu de complementos

Puede agregar hasta 9 herramientas que pueden ejecutarse cuando las necesite. Cada herramienta tiene su propia tecla de acceso rapido para su conveniencia. Tambien puede pasar los parametros a su herramienta. Para pasar el nombre del archivo abierto actual use una palabra especial $SB_FileName.

Codigo de comentarios

Los comentarios se denotan con una doble barra //, como se ve en otros lenguajes de scripting (php, etc.). Todo el texto colocado despues de // (dentro de esa misma linea) sera ignorado por el compilador.

Para comentar lineas multiples o parte de la linea, use comentarios de bloque {}:

0001: wait  {comments here} 0 ms

Desde la version 3.06, Sanny Builder tambien admite comentarios similares a C++./* */.

0001: wait  /* comments here */ 0 ms

Tambien es posible comentar lineas multiples seleccionandolas y presionando Ctrl+Q. Para descomentar las lineas, repitalo una vez mas.

Etiquetas personalizadas

Puede dar a las etiquetas nombres personalizados que no dependan de las opciones del descompilador. Los nombres se especifican en el archivo CustomLabels.ini (separadamente para cada juego).

Sintaxis del archivo:
<label offset> = <nombre personalizado>

Si el decompilador encuentra una etiqueta en el desplazamiento especificado, obtendra el nombre personalizado.

Para averiguar la compensacion de etiquetas, ajuste el formato de las etiquetas a "Compensacion global" en las opciones. Despues de descompilar, el numero en un nombre de etiqueta es su compensacion.

Comprobando actualizaciones

Sanny Builder puede verificar si hay una nueva version disponible. Para usar esta funcion, vaya al menu "Ayuda-> Buscar actualizaciones ...". El editor se conectara a los servidores de actualizacion y propondra descargar e instalar un archivo de actualizacion si este existe. El editor puede verificar automaticamente si hay una actualizacion disponible al inicio si habilita esta opcion en la ventana del actualizador.
