El operador IF evalúa una o más condiciones y crea una nueva rama de código.

Declaraciones IF de bajo nivel

Sintaxis
00d6: if <N>
    <1 condición>
    <2 condición>
    ...
    <N+1 condición>
004D: jump_if_false <etiqueta>
  • <N> - significa el número total de condiciones dentro de la declaración IF y la forma en que ocurre la evaluación de las condiciones.
  • N Número de condiciones Operador lógico
    0 1 la declaración IF es verdadera sí la condicion es verdadera
    1..7 2..8 AND (todas las condiciones deben ser verdaderas para que la declaración IF sea verdadera)
    21..27 2..8 OR (al menos una de las condiciones debe ser verdadera para que la declaración IF sea verdadera)
    Una sola instrucción IF puede contener hasta 8 condiciones.
    Sanny Builder permite omitir el 0 después del IF.
    IF 0 y IF son equivalentes.
  • <etiqueta> - el nombre de la etiqueta donde el script saltara si la declaración IF es falsa.
  • <condicíon> - cualquier opcode condicional que evalúe como True o False

  • Si usted tiene las condiciones de verificación habilitado en las opciones, se puede sustituir el número si con las palabras clave AND y OR. El compilador calcula el valor correcto por sí mismo.

    if and 
    $var1 > 0
    $var2 == 10.0
jf @dondequiera

  • El compilador escribe el número 1 en lugar de and.

  • IF AND - condiciones conectadas con el operador lógico AND (un reemplazo de if 1..7)
  • IF OR - condiciones conectadas con el operador lógico OR (un reemplazo para if 21..27)

Construcctor de alto nivel

Para facilitar las condiciones de escritura, existen construcciones de alto nivel que no requieren etiquetas adicionales:

Sintaxis
IF <N>/AND/OR
    <1 condición>
    <2 condición>
    ...
    <N+1 condición>
THEN
    <comandos si la declaración es verdadera>
END

IF <N>/AND/OR
    <1 condición>
    <2 condición>
    ...
    <N+1 condición>
THEN
    <comandos si la declaración es verdadera>
ELSE
    <comandos si la declaración es falsa>
END

Una condición es creada con las reglas descritas de las condiciones de bajo nivel.
Después de THEN, debe especificar los comandos que se ejecutarán si se cumple la condición.
Después de ELSE, debe especificar los comandos que se ejecutan si no se cumple la condición.

La opción de verificación de condiciones debe estar habilitada.
Se admiten declaraciones IF anidadas.

Operadores relacionales

A == B A es igual a B
A <> B A es desigual a B
A >= B A es mayor o igual que B
A <= B A es menor o igual que B
A > B A es mayor que B
A < B A es menor que B

A y B son operandores. El compilador puede averiguar un código de operación si uno de los operandos es un número o una cadena literal, o se conoce el tipo de ambas variables.
