Ovbimente si estas aca, es por que intentaste pasar bien tu mod de pc a android y no te funciono del todo bien.
Bueno, para pasar un mod de PC a ANDROID, hay varias opciones.

  1. Cambios de condiciones.
  2. Cambios a opcodes.
  3. Formato del script.

1. Cambio de condiciones
&

Si estas usando condiciones especiales de una PC, debes darte cuenta de que si colocas una condicion que por ejemplo, que evalue si precionamos una tecla, es estupido.

wait 0
if
  ifPressed.PhysicalKey(TAB)
else_jump @myLabel
wait 1000
...
En este caso, la mejor opcion es cambiar de teclado fisico a virtual.
El teclado virtual solo lee las acciones que realizas como "correr, agacharte, caminar, etc".
Por ejemplo, aqui, en vez de decirle si precionamos "tabulador", ponemos si presionamos "accion": 
wait 0
if
   ifPressed.VirtualKey(0, KEY_OF_ACTION)
else_jump @myLabel
wait 1000
...


2. Cambio a opcode
&

Este paso se da mas, cuando se usa clases, ya que ya mayoria de opcodes de PC no son los mismo que en Android, y muchas veces no tenemos los mismos opcodes, asi que debemos pasar todas las clases a opcodes y comprobar de que sean las mismas.

wait 0
if
   ifPosition.ActorAtXYZ($PLAYER_ACTOR...)
else_jump @myLabel
wait 1000
asActor.WalkTo(1@...)
...
Esto, pasa a esto: 
0001: wait 0 ms
00D6: if
00fe:   actor $PLAYER_ACTOR...
004D: jump_if_false @myLabel
0001: wait 1000
07CD: AS_actor 1@... 
...
Tambien, puedes borrar los comentarios y espacios que estan entre los parametros, dejandolo asi (esto ultimo es opcional): 
0001: 0
00D6:
00fe: $PLAYER_ACTOR...
004D: @myLabel
0001: 1000
07CD: 1@... 
...

Esto es algo muy comun, algo que le suele pasar a mucho, y la pregunta es:
¿Que tipo de extencion le tengo que dar?
Bueno, eso dependera para tu mod, si deseas que se active solo siempre que realice sus condiciones, o que se active todo atra vez del menu de cleos.

Formato Descripcion
.CSA Es para mods que se activan de forma automatica al hacer alguna accion o al precionar algo, o que simplemente el mod ya se activa al iniciar una partida.
